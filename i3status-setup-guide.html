Dotfiles i3status Setup Notes

Pattern: scripts refresh cache files, i3status reads those files, and systemd --user owns recurring work.

1. Add the script

~/dotfiles/scripts/my-status-script
chmod +x ~/dotfiles/scripts/my-status-script
ln -sfn ~/dotfiles/scripts/my-status-script ~/.local/bin/my-status-script

2. Write to a cache file

mkdir -p "$HOME/.cache/i3status"
printf 'ICON value\n' > "$HOME/.cache/i3status/my-status"

3. Add a user service and timer

~/dotfiles/systemd/my-status.service
~/dotfiles/systemd/my-status.timer
ln -sfn ~/dotfiles/systemd/my-status.service ~/.config/systemd/user/my-status.service
ln -sfn ~/dotfiles/systemd/my-status.timer ~/.config/systemd/user/my-status.timer
systemctl --user daemon-reload
systemctl --user enable --now my-status.timer

4. Read it from i3status

order += "read_file my_status"

read_file my_status {
    path = "/home/jm/.cache/i3status/my-status"
    color_good = "#7AA2F7"
}

Rule of thumb

Do not start polling scripts from i3/config. Keep i3 focused on window/session behavior, keep recurring refresh work in systemd --user, and keep bar display state in ~/.cache/i3status.
